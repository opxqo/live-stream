# 分布式直播 - 本地专属信号推流客户端

这是一个轻量级、无依赖的跨平台桌面小工具（基于 Python Tkinter 构建）。
专门为您配合服务端的分布式直播系统量身定制。它能够一键捕获您本地电脑的硬件摄像头或桌面屏幕，并通过超低延迟编码发往服务端的 MediaMTX 中转站，由主程序接力融合为“画中画”。

## 使用前提
1. 已安装 **Python 3.8+**。
2. 电脑上已安装并配置好 **FFmpeg** 的系统环境变量（客户端依赖 FFmpeg 抓屏及编码推流）。

## 如何启动
双击运行，或在终端输入：
```bash
python main.py
```

## 功能说明
- **服务器配置**：填入您云服务器的公网 IP，默认端口 1935。
- **推流鉴权**：系统已写死默认账号与您服务器上的安全锁 (`liveadmin` : `L1ve!P@ss`) 一脉相承。
- **本地视频源**：
    - `webcam` 模式：程序会在后台自动扫描出您电脑连接的所有摄像头让您挑选（支持免驱 UVC。基于 windows dshow 协议）。
    - `desktop` 模式：全屏幕抓捕您的桌面（无声），由于不消耗算力适合作为无限制挂机信号源发往网络。
- **编码特性**：已在底层参数组中锁死 `preset ultrafast` 与 `tune zerolatency` 的极低延迟画幅配置，推流比特率预设为稳定且清晰的 `1500kbps`，即便网络存在颠簸，服务端主程序也不会报错挂死。
